# 架构与功能说明

本文记录当前系统的稳定架构事实，供后续维护、验收和部署判断使用。

## 总体架构

系统是单体 Web 应用：

- 后端：FastAPI + SQLAlchemy + SQLite。
- 前端：React + TypeScript + Vite。
- 图谱可视化：Cytoscape.js。
- LLM：MiniMax OpenAI-compatible Chat Completions；测试时可用 `APP_DISABLE_LLM=true` 禁用外部调用。
- 部署：Docker Compose，当前 VPS 由宿主机 Caddy 将 `/api/*` 反代到后端，其余路径反代到前端。

核心请求流：

1. 用户在问答工作台提交中文问题。
2. 后端进行多轮上下文化、课程范围判断和教材检索。
3. 检索返回 3-5 条教材文本块，知识图谱返回相关概念子图和推荐问题。
4. LLM 根据教材证据、图谱关系和最近对话生成中文回答；无 Key 或测试模式走本地降级回答。
5. 回答、来源、图谱、推荐问题和性能统计保存到会话消息与历史记录。

## RAG 与检索

主语料仍是 `ISLRv2_corrected_June_2023.pdf`。中文扫描教材不作为默认 RAG 输入。

当前支持可切换检索模式：

- `tfidf`：使用本地 token、IDF、章节加权和中文术语英译扩展。
- `vector`：使用本地持久化哈希词向量索引，索引文件保存在 `data/vector_index/`。
- `hybrid`：合并向量召回和 TF-IDF 结果，按合并分数排序。
- `auto`：向量索引可用时走向量检索，否则自动降级到 TF-IDF。

配置项：

- `APP_RETRIEVAL_MODE`：`tfidf`、`vector`、`hybrid` 或 `auto`。
- `APP_VECTOR_INDEX_DIR`：向量索引目录，默认 `data/vector_index`。
- `APP_REFERENCE_BOOKS_DIR`：管理员上传参考书目录，默认 `data/reference_books`。
- `APP_MAX_REFERENCE_BOOK_MB`：上传 PDF 大小上限。

向量索引实现位于 `backend/app/vector_index.py`。它使用确定性的哈希词向量和 L2 归一化，不依赖外部 embedding API，优先保证弱 VPS 可构建、可部署、可降级。题目要求的“向量数据库/向量索引路径”由该模块承担，后续可替换为 FAISS 或 Chroma，但必须保留相同的检索接口和 fallback 语义。

可信度守门逻辑仍在 `backend/app/rag.py`：

- 清晰无关问题返回 `out_of_scope`，不展示来源、推荐问题和图谱。
- 证据不足问题返回 `insufficient_evidence`，不能编造来源。
- 每轮追问都重新计算来源、推荐问题和图谱。

## 教材与索引管理

参考书元数据保存在 `reference_books` 表。默认启动时会为 `APP_PDF_PATH` 创建一条参考书记录，并设为当前启用教材。

管理员后台提供：

- 查看参考书列表和当前教材。
- 上传新的 PDF 参考书。
- 切换当前参考书。
- 重建文本块和向量索引。
- 查看索引状态、文本块数量、向量索引是否可用。

上传文件只保存到受控运行数据目录，不允许管理员输入任意服务器绝对路径。上传教材、SQLite 数据库和向量索引都属于运行数据，不提交 Git。

## 数据模型

主要表：

- `users`、`session_tokens`：用户、角色和登录会话。
- `chat_conversations`、`chat_messages`：多轮问答会话。
- `question_history`：历史记录，保存当时问题、回答全文和来源。
- `favorites`、`user_notes`：收藏和个人笔记。
- `concepts`：知识点。
- `graph_edges`：知识点关系边。
- `qa_pairs`：课程问答对。
- `reference_books`：RAG 参考书元数据。
- `index_jobs`：索引任务记录预留表。
- `text_chunks`：教材文本块，包含章节、页码、来源文件、索引模型和可选 `book_id`。
- `system_config`：运行时 LLM 配置。

当前没有 Alembic，新增字段和表通过 `backend/app/database.py` 中的兼容迁移创建。改数据模型时必须考虑线上 SQLite 已有数据。

## 管理后台

前端管理后台已拆为总览和子模块：

- 总览：系统规模、当前教材、检索模式、向量索引状态。
- 教材与索引：参考书上传、切换、重建。
- 用户与权限：角色、改密、注销、删除。
- 模型 API：Base URL、模型、API Key 保存与检查。
- 问答对：跳转到分页管理页。
- 文本块：搜索和重建。
- 知识点：CRUD。
- 图谱边：CRUD。
- 历史审计：查看提问摘要。

普通学生不能看到管理后台和问答对管理入口；后端写入型管理接口均依赖 `require_admin`。

## 前端页面

- 登录/注册页。
- 问答工作台：会话侧栏、问题输入、回答、来源、推荐问题、图谱子图、性能条、收藏。
- 问答对管理：搜索、类型筛选、分页、新增、编辑、删除。
- 知识图谱：概念搜索、子图展示、节点点击、推荐问题。
- 我的笔记：收藏回答、全部历史、回答全文、来源和个人笔记。
- 管理后台：模块化后台控制台。
