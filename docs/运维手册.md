# 运维手册

本文面向后续维护者和 AI Agent，说明线上站点如何运行、如何检查、如何备份、如何发布新版本，以及出问题时如何回滚。

- VPS：`165.232.60.232`
- SSH 主机别名：`blues-vps`
- 线上目录：`/opt/advanced-stat-qa/app`

```caddy
aistudyassistant.bluesclawd.dev {
    encode gzip zstd
    reverse_proxy /api/* 127.0.0.1:18000
    reverse_proxy 127.0.0.1:18080
}
```

## 线上敏感配置

线上 `.env` 文件位于：

```bash
/opt/advanced-stat-qa/app/.env
```

它包含：

- `MINIMAX_BASE_URL`
- `MINIMAX_MODEL`
- `MINIMAX_API_KEY`
- `APP_DATABASE_URL`
- `APP_PDF_PATH`
- `APP_RETRIEVAL_MODE`
- `APP_REFERENCE_BOOKS_DIR`
- `APP_VECTOR_INDEX_DIR`
- `APP_MAX_REFERENCE_BOOK_MB`
- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- 其他 Docker/应用配置

要求：

- `.env` 不提交 Git。
- `.env` 不放进镜像层。
- 文件权限保持 `600`。
- 不在截图、报告、README、聊天回复中复述真实 API Key。

查看权限：

```bash
ls -l /opt/advanced-stat-qa/app/.env
```

修正权限：

```bash
chmod 600 /opt/advanced-stat-qa/app/.env
```

## 健康检查

服务器本机检查：

```bash
curl -fsS http://127.0.0.1:18000/api/health
curl -I -s http://127.0.0.1:18080 | head
```

公网检查：

```bash
curl -fsS https://aistudyassistant.bluesclawd.dev/api/health
curl -fsS https://aistudyassistant.bluesclawd.dev/api/system/stats
```

预期：

```json
{"status":"ok","app":"高级统计方法智能问答系统"}
```

系统统计应大致为：

```text
users >= 1
concepts = 69
graph_edges = 336
qa_pairs = 70
text_chunks = 1413
pdf_available = true
retrieval_mode = auto
active_book.filename = ISLRv2_corrected_June_2023.pdf
```

`vector_index_ready` 表示 `data/vector_index/` 下是否存在可用本地向量索引。当前后端在 `auto`、`vector` 或 `hybrid` 模式下启动时会尝试自动补齐缺失索引；更换教材或自动补齐失败时，可在管理后台“教材与索引”中点击重建。

## 数据持久化

线上使用 Docker named volume 保存 SQLite 数据库和运行数据。不要以为重建镜像会删除业务数据；只要不删除 volume，用户、历史、会话、知识点、图谱边、问答对、文本块、参考书上传文件和向量索引都会保留。

查看 volume：

```bash
docker volume ls | grep advanced-stat-qa
```

后端容器内数据库路径：

```text
/app/data/app.db
```

## 备份

推荐先停服务，再备份 Docker volume 中的 `data/`：

```bash
cd /opt/advanced-stat-qa/app
docker compose -p advanced-stat-qa -f docker-compose.server.yml down
mkdir -p /opt/advanced-stat-qa/backups
docker run --rm \
  -v advanced-stat-qa_app_data:/data:ro \
  -v /opt/advanced-stat-qa/backups:/backup \
  alpine sh -c 'cd /data && tar -czf /backup/app-data-$(date +%F-%H%M%S).tar.gz .'
docker compose -p advanced-stat-qa -f docker-compose.server.yml up -d
```

查看备份：

```bash
ls -lh /opt/advanced-stat-qa/backups
```

## 从本地发布新版本

### 自动部署（推荐）

本项目已配置 GitHub Actions 自动部署。每次 `git push` 到 `main` 分支时，GitHub Actions 会自动完成打包、上传、构建和启动。

本地只需执行：

```powershell
.\scripts\verify.ps1
git add .
git commit -m "说明本次修改"
git push
```

GitHub Actions 会自动完成后续部署，详见 `.github/workflows/deploy.yml`。

### 手动部署（备用）

如果自动部署不可用，可以手动发布：

本地修改代码后，先完整验证：

```powershell
.\scripts\verify.ps1
```

提交并推送 GitHub：

```powershell
git status
git add .
git commit -m "说明本次修改"
git push
```

打包源码并上传到 VPS。注意排除 `.env`、虚拟环境、数据库、本地构建产物和依赖目录：

```powershell
tar -czf "$env:TEMP\advanced-stat-qa.tar.gz" `
  --exclude=.git `
  --exclude=.env `
  --exclude=.venv `
  --exclude=data `
  --exclude=frontend/node_modules `
  --exclude=frontend/dist `
  --exclude=__pycache__ `
  --exclude=.pytest_cache `
  .

scp "$env:TEMP\advanced-stat-qa.tar.gz" blues-vps:/tmp/advanced-stat-qa.tar.gz
```

在服务器上发布：

```bash
ssh blues-vps
cd /opt/advanced-stat-qa

docker compose -p advanced-stat-qa -f app/docker-compose.server.yml down

rm -rf app.new
mkdir app.new
tar -xzf /tmp/advanced-stat-qa.tar.gz -C app.new

cp app/.env app.new/.env
chmod 600 app.new/.env

rm -rf app.prev
mv app app.prev
mv app.new app

cd app
docker compose -p advanced-stat-qa -f docker-compose.server.yml up -d --build
```

发布后验收：

```bash
curl -fsS http://127.0.0.1:18000/api/health
curl -fsS https://aistudyassistant.bluesclawd.dev/api/health
curl -fsS https://aistudyassistant.bluesclawd.dev/api/system/stats
```

浏览器检查：

1. 打开 `https://aistudyassistant.bluesclawd.dev`。
2. 管理员登录。
3. 问答工作台提问“我是谁”，确认没有来源和图谱。
4. 提问“岭回归和 Lasso 有什么区别？”，确认有来源和图谱。
5. 打开知识图谱，搜索“岭回归”。
6. 打开管理后台，确认模型配置、用户管理、文本块和图谱数据正常。

## 回滚

如果新版本上线后有严重问题，并且上一次版本保存在 `/opt/advanced-stat-qa/app.prev`：

```bash
ssh blues-vps
cd /opt/advanced-stat-qa

docker compose -p advanced-stat-qa -f app/docker-compose.server.yml down || true

rm -rf app.bad
mv app app.bad
mv app.prev app

cd app
docker compose -p advanced-stat-qa -f docker-compose.server.yml up -d --build
```

回滚后同样执行健康检查。

## Caddy 运维

验证 Caddy 配置：

```bash
caddy validate --config /etc/caddy/Caddyfile
```

重载 Caddy：

```bash
caddy reload --config /etc/caddy/Caddyfile
```

查看 Caddy 是否占用 80/443：

```bash
ss -tulpn | grep -E ':(80|443) '
```

如果公网打不开，但本机端口正常，优先检查：

1. DNS 是否仍指向 VPS。
2. Caddy 是否运行。
3. `/etc/caddy/Caddyfile` 是否包含站点块。
4. 防火墙是否开放 80/443。

## 管理员账号

公网默认管理员密码已经不再是 `Admin@123456`。如果需要查看当前种子密码：

```bash
grep '^ADMIN_' /opt/advanced-stat-qa/app/.env
```

更推荐登录管理后台后修改密码，或通过后台用户管理功能为学生重置密码、注销会话、删除账号。

## MiniMax 与模型配置

有两层配置来源：

1. `.env`：启动时的默认配置。
2. 数据库 `system_config` 表：管理员后台保存的运行时配置。

管理员后台可修改：

- Base URL
- 模型名称
- API Key

后端接口：

```text
GET   /api/admin/llm-config
PATCH /api/admin/llm-config
POST  /api/admin/llm-config/test
```

接口不会返回完整 Key，只返回是否已配置和脱敏预览。

## 常见故障

### 公网首页打不开

检查：

```bash
curl -I http://127.0.0.1:18080
curl -fsS http://127.0.0.1:18000/api/health
caddy validate --config /etc/caddy/Caddyfile
```

如果本机端口正常，多半是 Caddy、DNS 或防火墙问题。

### API 返回 502

通常是后端容器没起来：

```bash
cd /opt/advanced-stat-qa/app
docker compose -p advanced-stat-qa -f docker-compose.server.yml ps
docker compose -p advanced-stat-qa -f docker-compose.server.yml logs --tail=120 backend
```

### 问答很慢

先看前端回答里的性能条。如果 `llm_ms` 很高，主要慢点在 MiniMax 生成；如果 `retrieval_ms` 很高，检查文本块索引和数据库状态。

### 文本块数量为 0

检查 PDF 是否存在：

```bash
docker compose -p advanced-stat-qa -f docker-compose.server.yml exec backend ls -lh /app/ISLRv2_corrected_June_2023.pdf
```

然后在管理后台重建文本块索引，或重启后端观察日志。

### 向量索引未就绪

检查系统统计：

```bash
curl -fsS https://aistudyassistant.bluesclawd.dev/api/system/stats
```

如果 `vector_index_ready=false`，先查看后端日志确认启动自愈是否报错；再用管理员登录后台，打开“教材与索引”，对当前参考书执行“重建”。重建会重新抽取文本块并写入 `data/vector_index/`。

### 管理后台看不到 LLM 已配置

检查 `.env` 和后台配置：

```bash
grep '^MINIMAX_' /opt/advanced-stat-qa/app/.env
```

不要把 Key 打印到公开渠道。

## 维护红线

- 不要提交 `.env`。
- 不要删除 Docker volume，除非已经确认备份。
- 不要在公网保留默认管理员密码。
- 不要把服务器 `.env` 覆盖成本地 `.env.example`。
- 发布新版本前必须先跑本地 `.\scripts\verify.ps1`。
- 发布后必须跑公网健康检查和至少一轮课程问答。
