基于高级统计方法课程知识图谱的智能问答系统设计与实现

1. 项目背景与目标

《高级统计方法》课程涉及统计学习、回归、分类、重采样、正则化、树模型、支持向量机和无监督学习等大量概念。学生在学习中常见的问题包括“某个概念是什么意思”“两个方法有什么区别”“某个方法适合什么场景”。传统教材检索需要人工翻页，效率低，也难以展示概念之间的结构关系。

本项目实现一个中文智能问答系统。系统结合教材文本检索、知识图谱和大模型回答生成，为学生提供概念解释、关系查询、应用场景说明、教材来源引用、图谱子图和学习历史记录。

2. 需求分析

系统覆盖题目要求的四类模块：

  • 问答对数据管理模块：内置 60 个以上高质量问答对，覆盖概念解释类、关系查询类和应用场景类，并提供增删改查、搜索和类型筛选。
  • RAG 问答模块：抽取教材 PDF 文本，按章节和页码切分为 200 个以上文本块；用户输入中文问题后检索 3-5 个相关文本块，并结合大模型生成中文回答。
  • 知识图谱辅助问答模块：构建 60 个以上课程知识点和 100 条以上关系边，支持节点点击推荐问题、问答结果页展示局部子图。
  • 用户管理模块：实现注册、登录、登录状态显示、个人提问历史查看和清空。

3. 教材与语料选择

项目目录中有英文版 ISLRv2_corrected_June_2023.pdf 和中文版 [BOOK]统计学习导论 基于R应用-高清中文.pdf。经过抽样检查，英文 PDF 为文本型文件，绝大多数页面可直接抽取正文；中文 PDF 为扫描图片型文件，页面缺少可抽取文字层。

因此系统选择英文 PDF 作为 RAG 主语料。这样可以稳定保留正文、章节和页码引用。中文学习体验通过中文界面、中文问答对、中文术语扩展和中文回答生成实现。

4. 系统架构

系统采用前后端分离架构：

  • 前端使用 React、TypeScript 和 Vite，提供问答工作台、问答对管理、知识图谱、学习历史四个页面。
  • 后端使用 FastAPI 提供 REST API。
  • 数据库使用 SQLite，通过 SQLAlchemy 管理用户、会话、问答对、知识点、图谱边、文本块和历史记录。
  • 检索模块使用本地混合检索：对中文问题进行术语扩展，将“交叉验证、岭回归、逻辑回归”等中文表达映射到英文教材术语，再对教材文本块进行相关性评分。
  • LLM 模块默认适配 MiniMax OpenAI-compatible Chat Completions。未配置 API Key 时，系统会使用本地证据生成回答，保证演示可复现。

5. 数据设计

主要数据表包括：

  • users：保存用户账号和密码哈希。
  • session_tokens：保存登录会话。
  • concepts：保存知识点中文名、英文名、别名、章节和说明。
  • graph_edges：保存知识点之间的关系类型和证据。
  • qa_pairs：保存问题、答案、问题类型、关联知识点和来源。
  • text_chunks：保存教材文本块、章节、页码、来源文件和检索模型标识。
  • question_history：保存每个用户的提问、答案、摘要和来源。

种子数据覆盖统计学习核心章节，包括监督学习、无监督学习、线性回归、逻辑回归、判别分析、交叉验证、正则化、树模型、随机森林、提升法、支持向量机、PCA 和聚类等主题。

6. RAG 流程

系统的问答流程如下：

  1. 用户输入中文自然语言问题。
  2. 系统识别问题中出现的核心知识点和中文统计术语。
  3. 检索模块将中文术语扩展为英文教材术语，例如“交叉验证”扩展为 cross-validation、k-fold、leave-one-out。
  4. 系统在教材文本块中计算相关性得分，返回最相关的 3-5 个文本块。
  5. 系统查询知识图谱，生成以核心概念为中心的局部子图。
  6. 若配置了 MiniMax API Key，系统调用大模型生成中文回答；否则使用本地证据生成可解释回答。
  7. 回答末尾展示教材来源，包括文件名、章节、节名和 PDF 页码。
  8. 系统保存用户提问历史。

7. 知识图谱设计

知识图谱节点为课程知识点，边为知识点之间的课程关系。关系类型包括：

  • 包含：上位概念包含下位概念。
  • 属于：下位概念属于某个知识体系。
  • 相关：两个知识点在学习或应用中相关。
  • 区别：两个方法需要比较差异。
  • 用于：某个方法用于解决某类问题。
  • 前置：一个概念是理解另一个概念的前置条件。
  • 组成：一个概念是另一个方法的重要组成部分。

前端使用 SVG 绘制局部子图，节点显示中文名和英文名，边显示关系类型。点击知识点后，系统推荐三个相关问题，帮助学生继续学习。

8. 用户界面

系统界面以课程学习工具为定位，避免营销式页面。主要页面如下：

  • 登录/注册页：创建或进入个人账号。
  • 问答工作台：输入问题、查看回答、来源、相关问题和图谱子图。
  • 问答对管理页：查看、搜索、新增、编辑、删除问答对。
  • 知识图谱页：搜索知识点、查看局部图谱、获取推荐问题。
  • 学习历史页：查看和清空个人提问历史。

9. 测试与验证

项目提供 scripts/verify.ps1 作为统一验证入口。测试覆盖：

  • 系统健康检查与统计数量。
  • 用户注册和登录。
  • 问答对增删改查。
  • RAG 问答和来源引用。
  • 图谱子图和节点推荐问题。
  • 历史记录查看和清空。
  • 前端 TypeScript 构建。

推荐端到端演示问题包括：

  • 什么是交叉验证？
  • 岭回归和 Lasso 有什么区别？
  • 逻辑回归可以用来解决什么问题？
  • 偏差-方差权衡如何解释过拟合？
  • 随机森林和提升法在集成思路上有什么不同？
  • 为什么做 PCA 前通常要标准化？

10. 结论

本项目完成了题目要求的核心功能，并在最低要求基础上扩展了数据规模、图谱结构和端到端验证。系统能够以中文形式服务课程学习，同时利用英文教材的高质量文本层保证 RAG 检索和来源引用的可靠性。整体方案避免了过重的外部图数据库依赖，适合课程项目演示、复现和后续扩展。
